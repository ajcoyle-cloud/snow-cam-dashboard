Radar Overlay — Prototype

Warp a radar screenshot onto the 3D NZ terrain: crop the UI chrome, isolate rain/echo pixels from the map underneath, then drape the result geographically. Everything here runs client-side on one static image — swapping the file input for a polled network feed is the next step toward live animation.

3. Isolate rainfall pixels

Keeps only saturated pixels in the checked hue bands — filters out the dark basemap, white coastlines and pale label text. Tune the sliders until only the echo colour blobs survive in the preview below. The coastline checkbox keeps the source image's own thin white coastline stroke too (measured: sat ~3%, brightness 100% — under the saturation gate above) as a placement check against the real satellite coastline once draped.

4. Georeference

Matches the land shape in your crop (grey basemap pixels) against real NZ coastline data and searches for the bounds that fit best — no manual entry needed for a straight-on screenshot like this one.
Starting rectangle for auto-align, and a manual fallback. After draping, the 4 red corner pins on the map are draggable — drag any one directly to fine-tune the fit (independently, so it also corrects any skew).

5. Animate (live frames)

Fetches the last several available frames, runs each through the current crop/isolate settings, and lets you play them back draped on the map — the actual goal, not just a single static overlay.
